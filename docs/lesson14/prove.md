# W14 Prove: Final Project - Final Submission
## Instructions
Submit **one link to your GitHub repository** for your completed Playlist Final Project.

Submit the link to the **repository itself**, not just a single file or folder. Your repository should make it easy to find your three implementations, tests, and final report.

Before you submit, use the checklist below. For the full project description and timeline, see the [Final Project Overview](../lesson11/overview). For the official behavior rules and edge cases, see the [Playlist Final Project FAQ](../lesson11/prove-faq).

## Final Submission Checklist
Before you submit your repository link, make sure:

- `Playlist1` is complete and implements `IPlaylist`
- `Playlist2` is complete and implements `IPlaylist`
- `Playlist3` is complete and implements `IPlaylist`
- All three implementations keep the same external behavior
- Your functional tests cover all three implementations
- Your performance tests compare all three implementations using the same operations and playlist sizes
- Your repository includes an updated `report.md`
- Your final recommendation is clearly written and supported by your data
- You have addressed any useful feedback from peer review
- Your repository link works correctly and is the one you will submit in Canvas

## What Your Repository Should Contain
Your final repository should clearly include:

1. All three playlist implementations in `Playlist1`, `Playlist2`, and `Playlist3`
2. Functional unit tests that verify all three versions
3. Performance tests that compare all three versions fairly
4. A completed `report.md`

## What `report.md` Should Include
Your final `report.md` should include:

1. A short explanation of each implementation
2. A comparison table or chart for key operations
3. A discussion of trade-offs you observed
4. A clear recommendation for which implementation you would choose and why
